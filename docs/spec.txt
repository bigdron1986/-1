1. Назначение приложения
Автоматизированный сбор, хранение и визуализация данных термометрии в силосах с пшеницей для контроля динамики изменений и выявления критических точек.

2. Способ работы
Портативное Windows-приложение, не требующее установки. Запускается из одной папки, где лежит .exe файл и вспомогательные файлы (база данных, настройки).

Работает полностью локально, без доступа в интернет.

Отчеты добавляются пользователем вручную (перетаскиванием файла в окно или через кнопку "Обзор").

3. Источник данных
Файлы: Excel (.xlsx) с именем termo_ДД.ММ.ГГГГ.xlsx (например, termo_05.03.2026.xlsx). Дата отчета извлекается из имени файла.

Структура файла: Один лист, данные идут последовательными блоками по силосам, как показано в примере.

Название силоса: Ячейка в столбце A, содержащая текст "Силос X" (где X — идентификатор).

Подвески: Строки, следующие за названием силоса, где в столбце A указано "подвеска Y".

Датчики: Значения температур для подвески находятся в столбцах B-G (датчики 1-6).

Спецзначение: 71,2 интерпретируется как "ошибка/обрыв датчика" и не участвует в расчетах средней температуры, но отображается и подсвечивается как критическое событие.

4. Функциональные требования
4.1. Загрузка данных:

Возможность добавить один или несколько файлов через стандартный диалог выбора файлов (кнопка "Добавить отчет(ы)").

Возможность перетащить файл(ы) мышкой в окно программы (drag-and-drop).

При загрузке файла программа парсит его, извлекает все показания (силос, подвеска, датчик, температура, дата) и сохраняет их во внутреннюю базу данных.

Защита от дублирования: если файл с такой же датой уже был загружен, программа должна спросить, что делать (пропустить/заменить/создать копию).

4.2. Хранение данных:

Все исторические данные хранятся в файле базы данных (SQLite) внутри папки с программой. Это обеспечит быстрый доступ и компактность.

4.3. Просмотр и анализ данных (Интерфейс):
Главный экран должен быть максимально информативным и удобным.

Панель фильтров:

Выбор силоса (из списка). По умолчанию можно выбрать "Все силосы", но для детального просмотра — один конкретный.

Выбор подвески (для выбранного силоса).

Выбор диапазона дат (календарь) для отображения.

Кнопки быстрого выбора периода: "День", "3 дня", "Неделя", "Месяц", "Всё время".

Интерактивная таблица "Горячие точки":

Таблица, в которой отображаются только проблемные места за выбранный период.

Строка таблицы: Силос, Подвеска, Датчик, Дата, Температура.

Цветовое выделение:

Любой датчик с температурой > X°C (где X — порог по ГОСТу, скажем, 15°C, но нужно будет сделать это настраиваемым) подсвечивается красным.

Датчики со значением 71,2 (ошибка) подсвечиваются серым или оранжевым.

При клике на любую строку в таблице график внизу должен автоматически построиться для этого конкретного датчика (силос + подвеска + номер датчика), показывая историю его изменений.

График динамики температур:

Отображает изменение температуры во времени.

Пользователь может выбрать, что именно строить (например, через выпадающие списки):

Конкретный датчик: (силос 3а, подвеска 5, датчик 4).

Все датчики одной подвески: (силос 4б, подвеска 2) — на графике будет 6 линий.

Средняя температура по подвеске/силосу.

На графике должна быть возможность показать/скрыть отдельные линии (легенда).

Отображение тренда (дельты): Рядом с графиком или при наведении на точку показывать изменение температуры за сутки и за 3 дня. Например: "+2.3°C за 3 дня".

4.4. Экспорт:

Возможность сохранить текущий график как картинку (PNG, JPEG).

Возможность экспортировать данные из таблицы "Горячие точки" в Excel или PDF для отправки отчета.

5. Технические детали
Платформа: Windows 10.

Язык программирования: Python (как самый быстрый для разработки такого прототипа).

Упаковка: Использование PyInstaller для создания одного .exe файла (или папки с файлами).

База данных: SQLite (файл temperatures.db).
